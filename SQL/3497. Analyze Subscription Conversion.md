# 3497. Analyze Subscription Conversion

## 📝 Problem

### Table: UserActivity

| Column Name       | Type    |
|------------------|---------|
| user_id          | int     |
| activity_date    | date    |
| activity_type    | varchar |
| activity_duration| int     |

- `(user_id, activity_date, activity_type)` is the unique key.
- `activity_type` ∈ ('free_trial', 'paid', 'cancelled').
- Each row represents a user's activity on a specific date.

---

## 🎯 Task

1. Find users who **converted from free trial to paid subscription**
2. Calculate:
   - Average daily activity duration during **free trial**
   - Average daily activity duration during **paid subscription**
3. Round both averages to **2 decimal places**
4. Return results ordered by `user_id` ascending

---

# 💡 Intuition

We only want users who:

- Have at least one `free_trial` record  
- Have at least one `paid` record  

That means they **converted**.

Then:

- Compute average duration for `free_trial`
- Compute average duration for `paid`
- Join both results
- Order by `user_id`

---

# 🚀 Approach

1. Create a CTE for free trial averages.
2. Create a CTE for paid subscription averages.
3. Inner join both CTEs (ensures only converted users).
4. Round averages to 2 decimal places.
5. Sort by `user_id`.

---

# ⏱ Complexity

- **Time Complexity:** `O(n)`
- **Space Complexity:** `O(n)` (for grouping)

---

# 💻 Code

```mssql
/* Write your T-SQL query statement below */

WITH freeUsers AS (
    SELECT 
        user_id,
        ROUND(AVG(1.0 * activity_duration), 2) AS trial_avg_duration
    FROM UserActivity
    WHERE activity_type = 'free_trial'
    GROUP BY user_id
),

paidUsers AS (
    SELECT 
        user_id,
        ROUND(AVG(1.0 * activity_duration), 2) AS paid_avg_duration
    FROM UserActivity
    WHERE activity_type = 'paid'
    GROUP BY user_id
)

SELECT 
    f.user_id,
    f.trial_avg_duration,
    p.paid_avg_duration
FROM freeUsers f
INNER JOIN paidUsers p 
    ON f.user_id = p.user_id
ORDER BY f.user_id;
```

---

# ✅ Why This Works

- `INNER JOIN` ensures only users who have both trial and paid records.
- `1.0 * activity_duration` forces decimal division.
- `ROUND(..., 2)` ensures 2 decimal precision.
- Separate CTEs make logic clean and readable.
