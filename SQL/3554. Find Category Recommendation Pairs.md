# 3554. Find Category Recommendation Pairs  
## Method 1 — Double Subquery Self Join

---

## Intuition

For each user, determine which product categories they have purchased from.  
Then generate all possible category pairs `(category1 < category2)` per user.  

Count how many unique users purchased from both categories.  
Only keep category pairs where at least **3 different users** purchased both.

---

## Approach

1. Join `ProductPurchases` with `ProductInfo` to get `(user_id, category)`.
2. Create two identical derived tables.
3. Self-join them on `user_id`.
4. Use `category1 < category2` to:
   - Avoid duplicate pairs like (A,B) and (B,A)
   - Avoid same-category pairs.
5. Use `DISTINCT` to ensure each user is counted once per category pair.
6. Group by category pairs.
7. Filter using `HAVING COUNT(*) >= 3`.
8. Order by:
   - `customer_count DESC`
   - `category1 ASC`
   - `category2 ASC`

---

## Complexity

- **Time Complexity:** O(n²) worst case (due to self join)
- **Space Complexity:** O(n)

---

## Code

```mssql
WITH cte AS (
    SELECT DISTINCT 
        category1, 
        category2, 
        c1.user_id
    FROM (
        SELECT 
            u.user_id, 
            p.category AS category1
        FROM ProductPurchases u
        INNER JOIN ProductInfo p 
            ON p.product_id = u.product_id
    ) c1
    INNER JOIN (
        SELECT 
            u.user_id, 
            p.category AS category2
        FROM ProductPurchases u
        INNER JOIN ProductInfo p 
            ON p.product_id = u.product_id
    ) c2
        ON c1.user_id = c2.user_id
    WHERE category1 < category2
)

SELECT 
    category1,
    category2,
    COUNT(*) AS customer_count
FROM cte
GROUP BY category1, category2
HAVING COUNT(*) >= 3
ORDER BY 
    customer_count DESC,
    category1 ASC,
    category2 ASC;
```

## Method 2 — Optimized Distinct Mapping

---

## Intuition

Instead of building the joined dataset twice, first create a distinct `(user_id, category)` mapping.  

Then self-join this smaller dataset to generate category pairs efficiently.

Because we ensure distinct `(user_id, category)` pairs first,  
we can safely use `COUNT(*)` without `COUNT(DISTINCT user_id)`.

---

## Approach

1. Build distinct `(user_id, category)` mapping.
2. Self-join on `user_id`.
3. Use `category1 < category2` to avoid duplicate/reverse pairs.
4. Group by category pairs.
5. Filter using `HAVING COUNT(*) >= 3`.
6. Order by required conditions.

---

## Complexity

- **Time Complexity:** O(n + k² per user)
- **Space Complexity:** O(n)

---

## Code

```mssql
WITH user_categories AS (
    SELECT DISTINCT
        pp.user_id,
        pi.category
    FROM ProductPurchases pp
    JOIN ProductInfo pi
        ON pp.product_id = pi.product_id
)

SELECT 
    uc1.category AS category1,
    uc2.category AS category2,
    COUNT(*) AS customer_count
FROM user_categories uc1
JOIN user_categories uc2
    ON uc1.user_id = uc2.user_id
    AND uc1.category < uc2.category
GROUP BY 
    uc1.category,
    uc2.category
HAVING COUNT(*) >= 3
ORDER BY 
    customer_count DESC,
    category1 ASC,
    category2 ASC;
```
