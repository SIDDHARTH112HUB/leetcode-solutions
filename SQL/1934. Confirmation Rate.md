# 1934. Confirmation Rate

## 📝 Problem

### Table: Signups

| Column Name | Type     |
|------------|----------|
| user_id    | int      |
| time_stamp | datetime |

- `user_id` is unique.
- Each row contains the signup time for a user.

---

### Table: Confirmations

| Column Name | Type     |
|------------|----------|
| user_id    | int      |
| time_stamp | datetime |
| action     | ENUM     |

- `(user_id, time_stamp)` is the primary key.
- `action` is either `'confirmed'` or `'timeout'`.

---

## 🎯 Task

The **confirmation rate** of a user is:

```
number of 'confirmed' messages / total confirmation requests
```

- If a user made **no confirmation requests**, the rate is `0`.
- Round the result to **2 decimal places**.
- Return the result in any order.

---

# 💡 Intuition

We need:

1. All users from `Signups`
2. Count total confirmation requests per user
3. Count how many were `'confirmed'`
4. Divide confirmed count by total requests
5. Handle users with **no confirmations** (rate = 0)

Since some users may not exist in `Confirmations`, we must use a **LEFT JOIN**.

---

# 🚀 Approach

- Use `LEFT JOIN` from `Signups` to `Confirmations`
- Use `SUM(CASE WHEN action = 'confirmed' THEN 1 ELSE 0 END)` to count confirmed
- Use `COUNT(c.action)` to count only existing confirmation rows
- Use `CASE` to handle division by zero
- Use `ROUND(..., 2)` for formatting

---

# ⏱ Complexity

- **Time Complexity:** `O(n)`
- **Space Complexity:** `O(1)` (excluding output)

---

# 💻 Code

```mssql
/* Write your T-SQL query statement below */

SELECT 
    s.user_id,
    ROUND(
        CASE 
            WHEN COUNT(c.action) = 0 THEN 0
            ELSE 1.0 * SUM(CASE WHEN c.action = 'confirmed' THEN 1 ELSE 0 END) 
                 / COUNT(c.action)
        END
    , 2) AS confirmation_rate
FROM Signups s
LEFT JOIN Confirmations c 
    ON s.user_id = c.user_id
GROUP BY s.user_id;
```

---

## ✅ Why This Works

- `LEFT JOIN` ensures users with no confirmation requests are included.
- `COUNT(c.action)` avoids counting NULL rows.
- `CASE WHEN COUNT(c.action) = 0` prevents division by zero.
- `1.0 *` forces floating-point division.
- `ROUND(..., 2)` ensures two decimal precision.

---

### 🔥 Example Output

| user_id | confirmation_rate |
|---------|-------------------|
| 6       | 0.00              |
| 3       | 0.00              |
| 7       | 1.00              |
| 2       | 0.50              |
